# 策维（Ceway）v1.3 开发计划

## 核心目标

v1.3 不扩散新场景，不急于引入 AI 或外部数据。

v1.3 的核心是完成 Decision Pipeline：

历史开奖 -> 趋势计算 -> 号码评分 -> 预算约束 -> 组合生成 -> 决策解释 -> 投注方案。

## P0 必须完成

### 1. 号码评分模块

建立“走势图 -> 综合评分 -> 推荐号码”的完整链路。

要求：

- 评分表支持排序。已实现：综合分、热度分、遗漏分、均衡分排序。
- Top 号码可展开为 Top20。已实现：顶部 Top 号码卡片点击展开。
- 趋势图 hover 时能显示热度、遗漏和评分。已实现：冷热号图表 tooltip 显示出现次数、遗漏、综合分和排名。
- 推荐号码必须能追溯到评分来源。已实现：评分解释进入表格和推荐方案解释。

### 2. 预算驱动组合生成器

根据预算自动选择组合结构，而不是固定推荐。

策略模式：

- 保守：低覆盖，优先控制注数。
- 平衡：默认模式，兼顾覆盖和预算。
- 激进：高覆盖，在预算内扩大拖码范围。

要求：

- 输入预算后自动筛选费用不超过预算的方案。已实现。
- 输出胆码、拖码、后区、注数、费用。已实现。
- 显示为什么选择该结构。已实现：方案 reason 字段说明策略、费用占比和综合评分。

### 3. 资金管理状态机

真正实现 Anti-Martingale 资金模型。

字段：

- 本金。
- 当前余额。
- 当前盈利。
- 当前注数级别。
- 上期奖金。
- 最大回撤。

规则：

- 未中奖：1 注。
- 中奖 5 元：2 注。
- 中奖 10 元：4 注。
- 未盈利或中断盈利：恢复 1 注。

已实现：Dashboard 支持输入本金、当前余额、当前级别和上期奖金，后端返回余额、盈利、投入、奖金、回撤和状态转移说明。

## P1 体验升级

### 4. 决策解释

每个推荐号码需要解释来源。

示例：

- 07：热度靠前，综合评分高。
- 18：热度稳定，结构平衡。
- 34：遗漏适中，补充大小比例。

已实现：评分表和组合方案展示号码解释；组合方案展示策略选择原因。

### 5. 历史推荐记录

保存推荐时间、预算、模式、方案、费用和解释。

已实现：`GET /records/dlt` 与 `POST /records/dlt`，当前使用 JSON 文件保存最近 100 条记录；前端展示最近 6 条。

### 6. 评分排序与筛选

支持按综合分、热度、遗漏、均衡分排序。

已实现：号码评分模块支持排序和 Top15/全部切换。

## P2 后续迭代

- SQLite 数据库存储。
- 回测模块。
- 多场景正式支持。

- AI。
- 舆情。

- 外部公开数据。
