# 策维 v1.12 正式使用版

## 版本定位

v1.12 是大乐透与双色球双场景的正式使用冻结版。选号只是入口，决策解释、组合管理、开奖复盘与风险提示是产品主线。

系统不预测开奖结果，不承诺提高中奖概率。历史评分、冷热、遗漏、奇偶、大小、和值和回测，只用于说明历史结构与辅助整理组合。

## 正式功能

- 大乐透与双色球完整历史开奖快照。
- 智能建议、纯随机、套餐模拟和人工选号。
- 单式、复式、胆拖；大乐透支持追加，单式支持倍投。
- 套餐中的单式与复式保持原票面结构，可整组保存。
- 已选方案保存、跨设备单用户云同步、待开奖状态和开奖后自动复盘。
- 实际奖级、命中号码数、奖金、成本、净收益和 ROI；奖金数据不完整时不展示伪精确收益。
- 最近投入、连续加码和资金暴露提示。

## 数据维护

- 大乐透历史从第 `07001` 期开始。
- 双色球历史从第 `2003001` 期开始。
- 本机 LaunchAgent 在开奖日多次检查 `78500.cn`，校验号码范围和期号连续性后更新 CSV、SQLite、GitHub 主分支和 Pages。
- 补抓任务会根据上一期日期推导下一合法开奖日；跳过检查不会覆盖最近一次失败记录。

## 使用边界

- 套餐活动有地区、期限和终端限制，必须以出票地销售终端显示为准。
- “智能建议”只代表当前历史评分规则下的组合，不代表未来概率优势。
- “纯随机”不使用历史评分，每个合法组合地位相同。
- 云同步采用固定内部账号，不提供公开注册；同步密码不得写入仓库。

## 发布验收

正式发布前必须通过：

1. 大乐透和双色球历史连续性校验。
2. 前端全部单元测试与生产构建。
3. 后端全部单元和 API 集成测试。
4. 套餐金额、复式展开、赠票、倍投、保存与复盘测试。
5. `390x844` 手机视口下两种彩票的生成、保存和复盘主流程。
6. GitHub Pages 实际网址加载和最新数据期号核对。

## 后续维护

只接受三类变更：数据维护、明确缺陷修复、真实使用中能降低操作成本的体验改进。快乐8、自定义场景、AI 预测、论坛舆情和外部关注度继续留在范围外。
