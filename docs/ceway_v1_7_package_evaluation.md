# 策维 v1.7 套餐评估说明

## 版本目标

v1.7 在决策风控闭环中增加“套餐评估”，用可核对的数据回答：

- 实际支付金额。
- 基础套餐展开注数。
- 终端确认后的赠票金额与赠票注数。
- 单注实际支付成本。
- 是否在用户本期总预算内。

## 当前数据口径

### 大乐透

展示 18、28、58、88 元四档标准套餐结构。不把已结束的派奖或未核实的地区赠票计入权益。

### 双色球

展示江苏福彩自第 2026080 期开始的套餐赠票活动。活动至 500 万元赠票资金用完为止，因此页面默认不计入赠票；只有用户确认当地终端明确显示赠票成功后，才计入对应权益。

## 计算口径

```text
票面展开注数 = 基础套餐展开注数 + 已确认赠票注数
单注实付成本 = 实际支付金额 / 票面展开注数
```

“票面展开注数”是组合数口径，不代表号码必然不重复，也不代表历史评分或单注中奖概率被提高。

## 风险边界

- 套餐评估只能在已设定的本期总预算内比较。
- 倍投倍赠会同比增加实际支出，不会继续降低单注实付成本。
- 不自动推荐“套餐 + 额外倍投”，避免活动成为追加预算的理由。
- 套餐未录入实际票面号码前，不能保存为待开奖推荐记录。
