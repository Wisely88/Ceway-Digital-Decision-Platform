# 策维 v1.8 验证闭环说明

## 目标

v1.8 不增加预测能力，重点补齐“生成、保存、开奖、复盘、风险解释”的产品闭环。

## 已交付

1. 系统建议一次生成主推、备选 A、备选 B 和单式兜底，可分别查看与保存。
2. 评分表号码可点击，展示最近出现期号、日期、遗漏、评分和方案入选状态。
3. DLT 与 SSQ 接入每期开奖奖级金额快照；只有数据完整时才计算净收益和 ROI。
4. 历史方案支持同步码导出与导入，数据包含在同步码内，不上传第三方。
5. Recharts 升级到 3.9.2，图表与静态演示数据改为按需加载。
6. 数据状态区展示开奖来源、快照构建时间、最新期号和数据质量。
7. DLT 与 SSQ 共享同一套交互和验证约束，保留各自独立的号码规则、评分和奖金数据。

## 验收口径

- 前端单元测试覆盖资金提示、套餐、号码轮换和同步码校验。
- 后端单元与 API 集成测试覆盖 DLT/SSQ 生成、保存、开奖复盘、奖级和实际奖金。
- GitHub Pages 生产构建无大包警告；首屏主 JS 约 294 KB，其余图表和演示数据按需加载。
- 390x844 手机视口验证场景进入、生成、保存、历史查看和复盘主流程。

## 风险边界

实际奖金只用于复盘历史结果，不作为未来收益推断。系统不预测开奖，不承诺中奖或回本，也不以历史 ROI 引导加码。
