# 策维 v1.6 决策风控说明

## 版本目标

v1.6 把产品主线固定为：帮助用户看清随机性、支出和风险，而不是制造可以战胜随机性的错觉。

完整流程：

```text
选择场景 -> 确认数据 -> 设置本期预算 -> 生成或人工选号
-> 查看决策体检 -> 保存 -> 等待开奖 -> 复盘 -> 检查长期行为
```

## 已交付范围

- 大乐透与双色球双场景。
- 完整历史 CSV 和 SQLite 数据底座。
- 冷热、遗漏、奇偶、大小、和值和号码评分。
- 单式、胆拖、预算约束和人工选号点评。
- 推荐期号、复制、保存、历史记录和开奖复盘。
- 历史滚动回测与随机选号对照。
- Anti-Martingale 状态机。
- 本期预算、近 30 日上限、本金暴露和连续加码提醒。
- GitHub Pages 静态演示，内置 DLT/SSQ 发布时数据快照。

## 决策体检口径

### 实际支出

以方案费用为准。单式和胆拖均按每注 2 元计算，费用不得高于本期预算。

### 覆盖与倍率

- 胆拖显示实际组合覆盖注数。
- 单式显示分散注数。
- 当前系统所有方案固定为 1 倍，不把多注覆盖描述为提高单注倍率。

### 本金暴露

```text
本金暴露 = 本期方案费用 / 初始本金
```

超过 2% 时进入关注，超过 5% 时提示暴露偏高。该阈值是行为控制线，不是收益模型。

### 周期投入

系统汇总最近 30 日已保存方案的真实费用。计入当前方案后超过用户设置的周期上限时，方案进入高风险状态。

### 连续加码

最近三次方案费用连续上升时触发追投提醒。高风险方案不能直接保存，用户需要先降低费用或回到基础预算。

### 长期表现

历史回测显示规则方案和随机对照在同一区间的历史命中记录率。它只用于检验规则是否稳定执行，不能解释为未来中奖率或收益率。

## 风险结论

- 低风险：预算内、周期上限内、未发现连续加码，单次资金暴露可控。
- 中风险：资金暴露或周期使用率接近控制线，不应提高倍率或临时追加。
- 高风险：超预算、超周期上限、连续加码，或未盈利时仍处于加注级别。系统建议暂停或缩减，并禁止直接保存。

## 验收命令

```bash
cd frontend
npm test
npm run build:pages

cd ..
backend/.venv/bin/python -m unittest discover -s backend/tests -v
```

验收要求：

- 前端决策风控测试全部通过。
- DLT/SSQ 生成器费用等于注数乘以 2 元，且不超过预算。
- 盈利后允许升级，未盈利立即回到基础注数。
- 静态演示构建成功，DLT 与 SSQ 均可进入。
- 数据最新期号、推荐期号和保存复盘期号一致。

## 明确边界

系统不提供中奖承诺、收益承诺、亏损追回、翻倍追投或“必出号码”。AI、论坛舆情和外部关注度不属于当前核心闭环。
